# SoulWriter AI 核心架构与提示词工程文档

本文档详细记录了 AI 重生爽文生成器中“章节生成/续写”与“章节重写”的功能原理及底层 Prompt 设计。

## 1. 章节正文生成/续写原理

### 核心逻辑
章节生成并非简单的“续写”，而是一个**受控的填充过程**。系统采用了“细纲驱动 + 上下文注入 + 风格化指令”的混合模式。

1.  **细纲驱动 (Blueprint Driven)**: AI 不是随意发散，而是必须严格遵循预先生成的 `ChapterOutline`（包含章节标题、梗概、爽点、叙事脉络）。
2.  **上下文注入 (Context Injection)**: 为了解决长文本遗忘问题，我们在每次生成时动态构建上下文窗口，包含：
    *   **世界观与人设**: 始终存在的静态约束。
    *   **长期记忆 (`storySummary`)**: 经过压缩的前文摘要，确保男女主关系和等级进度不崩。
    *   **短期记忆 (`prevContext`)**: 上一章的结尾内容，确保文本无缝衔接。
3.  **风格化 (Stylization)**: 通过 System Instruction 强制 AI 扮演“飞卢/起点爽文作家”，并规定感官描写方式。

### 核心提示词 (Prompt Template)

以下是 `services/geminiService.ts` 中 `streamChapterContent` 函数使用的 Prompt 模板：

```text
你现在是全网最火的爽文作家。请根据以下大纲撰写第 {{chapterNumber}} 章的正文。

【章节信息】
标题：{{title}}
梗概：{{summary}}
爽点：{{coolPoint}}
核心规则：{{rules}}

【前情提要 / 长期记忆】
(请参考此内容保持人设和剧情连贯，特别是男女主关系)
{{storySummary}}

【上下文】
上一章内容：{{prevContext}}

【世界与人物】
世界风格：{{world.theme}}
人物设定：
{{characterList_with_descriptions}}

【写作要求】
1. **文风**：小白文风格，短句为主，节奏极快。不要文青，不要谜语人。
2. **感官描写**：
   - 对女性角色：重点描写身材（长腿、细腰、曲线）、五官、声音（娇喘、冷哼）和气味（体香）。
   - 对打脸情节：描写反派的嚣张嘴脸（前）和震惊、恐惧的表情（后），形成强烈对比。
   - 对路人反应：要有大量的"倒吸一口凉气"、"目瞪口呆"等侧面描写来衬托主角的牛逼。
3. **对话**：要口语化，装逼台词要够硬，撩妹台词要够骚。
4. **字数**：2000字左右。
5. **排版**：段落之间空一行，方便手机阅读。
```

## 2. 章节重写 (Rewrite) 功能原理

### 核心逻辑
“重写”本质上是一次**带有修正指令的重新生成**。

1.  **状态回滚**: 当用户点击重写时，前端会清空当前章节的内容。
2.  **指令注入**: 系统复用上述的“章节生成”Prompt，但会在末尾追加一个【特别修订指令】区块。
3.  **定向修正**: AI 接收到原有的大纲和上下文，同时接收到了用户的修改意见（如“增加打斗细节”），从而在保持剧情走向不变的前提下，调整行文细节。

### 核心提示词 (Prompt Injection)

当用户输入重写指令（`additionalInstructions`）时，Prompt 会在【写作要求】后追加以下内容：

```text
【特别修订指令】
用户对本次写作有以下特别要求，请务必在文中体现：
{{additionalInstructions}}
```

*例如：用户输入“让反派死得更惨一点”，这段话就会被填入上述位置，强制 AI 执行。*

## 3. 后续章节推演 (Plot Extrapolation) 原理

### 核心逻辑
当用户点击“一键生成后续 N 章”时，系统采用了**“先回顾，后推演”**的自动工作流，并支持**人工干预**：

1.  **自动压缩 (Auto-Compression)**: 首先调用 `compressStoryHistory`，阅读当前所有已生成章节，提炼出最新的 `storySummary`（长期记忆）。
2.  **大纲规划**: AI 基于这个最新的记忆摘要和全局主线，进行逻辑外推，生成后续 N 章的细纲。
3.  **剧情定向 (Direction Control)**: 如果用户输入了“剧情走向控制”，该指令会被高优先级注入到 Prompt 中，强制 AI 将剧情引导至用户期望的方向（例如“主角进入遗迹”、“女主遭遇危险”）。
4.  **正文填充**: 基于新生成的细纲，串行生成正文内容。

### 核心提示词 (Prompt Template)

以下是 `generateNextChapterOutlines` 函数使用的 Prompt：

```text
你正在创作一部长篇爽文。目前已完成到第 {{lastChapterNumber}} 章。

【前情提要 / 长期记忆】
(这是之前所有剧情的浓缩摘要，请严格基于此规划后续剧情，保持连贯性)
{{storySummary}}

【当前进度】
上一章标题：{{lastChapterTitle}}
上一章剧情：{{lastChapterSummary}}

【主线大纲】
{{globalPlotSummary}}

【用户指定剧情走向（最高优先级）】
用户对接下来的剧情有明确要求，请务必将以下内容融入到接下来的 {{count}} 章细纲中：
>>> {{plotDirection}}

请继续规划接下来的 {{count}} 章（第 {{nextStart}} 章 到 第 {{nextEnd}} 章）的细纲。

要求：
1. **承上启下**：紧接上一章的剧情发展，不要断层。参考【前情提要】中的人际关系和伏笔。
2. **升级节奏**：主角的能力要逐步体现，遇到的敌人要越来越强（或者背景越来越深）。
3. **期待感**：每一章结尾都要留钩子。
4. **后宫互动**：如果没有打斗，就安排暧昧情节。
5. **指令执行**：必须响应用户的【指定剧情走向】要求，不要忽略。
```

## 4. 剧情压缩/长期记忆 (Memory Compression) 原理

### 核心逻辑
为了防止 Token 溢出导致 AI “变笨”或遗忘设定，我们引入了剧情压缩机制。

1.  **提取**: 抓取前 N 章的文本。
2.  **压缩**: 只要正文，去除格式噪音，如果太长则截断。
3.  **重组**: 要求 AI 以“网文编辑”的视角，将这些内容提炼为一份**设定集式的摘要**，而非简单的流水账。

### 核心提示词 (Prompt Template)

```text
你是一位网文编辑。请阅读以下小说前 {{endChapterIndex}} 章的内容，生成一份**详细的剧情回顾/记忆摘要**。
这份摘要将作为 AI 继续创作后续章节的"长期记忆"。

【重点关注人物】
{{protagonistNames}} (男主、女主列表)

【输入内容】
{{rawContent_of_previous_chapters}}

【输出要求】
1. **人称**：使用第三人称。
2. **核心剧情**：概括主角的等级变化、获得的关键宝物、击败的主要敌人。
3. **情感进度（至关重要）**：
   - 必须逐一列出主角与**每一位**已登场女主的互动进展。
   - 她们目前对主角的好感度如何？发生了什么暧昧或冲突事件？
4. **未完伏笔**：列出目前剧情中挖了坑但还没填的伏笔。
5. **格式**：分点叙述，条理清晰，字数控制在 1000 字以内。
```
