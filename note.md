# AI 重生爽文生成器 - 核心技术与业务流程说明

这份文档专为想要了解本项目底层逻辑的开发者（包括小白）编写。我们将从**技术原理**、**核心业务流程**、**AI 提示词（Prompt）大全**以及**调试指南**四个方面进行详细拆解。

---

## 1. 项目技术原理（小白秒懂版）

本项目是一个**纯前端**的 AI 写作应用。你可以把它想象成一条“自动化流水线”，AI 就是流水线上的工人，而代码负责把上一个工人的产出，精准地递给下一个工人。

### 1.1 核心技术栈
* **前端界面**：使用 React 19 构建，负责展示界面和接收用户点击。
* **样式美化**：使用 Tailwind CSS，通过简单的类名（如 `text-red-500`）快速画出好看的界面。
* **AI 大脑**：接入了 Google Gemini 和 阿里云通义千问（Qwen）的大模型 API。
* **数据记忆**：使用浏览器的 `localStorage`（本地存储），即使刷新网页，你写到一半的小说也不会丢。

### 1.2 关键原理解析
1. **链式生成（Prompt Chaining）**：
   AI 很难一次性写出 100 万字的小说。所以我们采用“拆解法”：
   `一句话脑洞 -> 世界观和角色 -> 整体大纲 -> 每章细纲 -> 单章正文`。
   每一步的输出，都会作为下一步的输入。
2. **结构化输出（JSON Schema）**：
   为了让代码能看懂 AI 的输出，我们强制 AI 按照特定的 JSON 格式返回数据（比如明确区分 `name` 和 `background`），而不是返回一堆乱七八糟的纯文本。
3. **长期记忆压缩（Context Compression）**：
   AI 的记忆（上下文窗口）是有限的，字数太多会变傻或报错。所以每写几章，系统会触发一个“记忆压缩”功能，让 AI 把前文总结成 1000 字以内的“剧情摘要”，带着这个摘要继续写后面的章节。

## 2. 核心业务处理流程

整个小说的生成就像是在玩通关游戏，分为以下几个核心步骤：

### 🟢 步骤 1：设定生成 (Setup -> Characters)
* **用户操作**：输入一句脑洞（例如：“重生回到十年前...”）。
* **系统处理**：调用 AI，根据脑洞生成**世界观**（力量体系、规则）和**角色列表**（男主、反派、后宫等）。

### 🟡 步骤 2：大纲生成 (Characters -> Plot)
* **用户操作**：确认或修改生成的角色和世界观。
* **系统处理**：把确认好的世界观和角色发给 AI，让 AI 规划出 4-6 个阶段的**主线大纲**（比如：微末崛起 -> 称霸学院 -> 征战万界）。

### 🟠 步骤 3：细纲生成 (Plot -> Outlines)
* **用户操作**：确认主线大纲。
* **系统处理**：AI 根据主线大纲，严格按照“黄金三章法则”（第一章受辱、第二章金手指、第三章打脸），生成前 N 章的**具体细纲**（包含每章的爽点、剧情梗概）。

### 🔴 步骤 4：正文写作 (Outlines -> Writing)
* **用户操作**：点击“生成正文”。
* **系统处理**：AI 结合世界观、角色、当前章细纲以及上一章的末尾内容，**流式（像打字机一样）** 生成 2000 字左右的正文。

### 🟣 步骤 5：记忆压缩与续写 (循环)
* **系统处理**：当现有细纲写完后，系统会将已写章节浓缩为“剧情摘要”。然后带着这个摘要，继续生成后续的细纲和正文，无限循环。

## 3. AI Agent 关键提示词 (Prompt) 大全

这里是项目中最核心的 AI 提示词模板。你可以直接复制这些提示词到 ChatGPT 或其他 AI 中测试效果。

### 3.1 角色与世界观生成
**作用**：根据一句话脑洞，裂变出完整的世界设定和人物关系。
```text
你是一位专门服务于中国年轻男性的网文大神，最擅长写"重生"、"都市/玄幻后宫"、"无敌流"爽文。
你的文风要骚气、热血、直白，深谙读者的爽点。

你的任务是根据用户的脑洞，设计一个极具吸引力的世界观和人物列表。

【核心要求】
1. **主角设定**：必须是男性，性格杀伐果断，不圣母，智商在线，拥有绝对逆天的金手指（系统、神器、前世记忆等）。
2. **后宫团设定**：必须至少设计 3-4 位不同类型的极品美女作为主要配角（如：高冷校花、傲娇大小姐、温柔青梅、妩媚女总裁、绝美师尊等）。
   - 每个人物都要有极其惊艳的外貌描写关键词（黑丝、长腿、御姐等）。
   - 每个人物都要有明显的"反差萌"或独特的性格标签。
   - 初始对主角态度可以是鄙视或路人，方便后续攻略。
3. **反派设定**：反派要嚣张跋扈，富二代或宗门圣子，专门用来给主角打脸，也就是"经验包"。
4. **世界观**：要有等级森严的制度，方便主角通过升级来打破规则，装逼打脸。

【用户脑洞】
{{prompt}}
```

### 3.2 主线大纲生成
**作用**：根据设定，规划小说的整体节奏。
```text
基于以下设定，生成一个让年轻男性读者热血沸腾的爽文主线大纲。

【世界设定】
{{worldContext}}

【人物设定】
{{charContext}}

要求：
1. **节奏紧凑**：分为 4-6 个关键大剧情节点。
2. **核心爽点**：每个阶段必须包含"扮猪吃虎"、"人前显圣"或"强势打脸"的情节。
3. **感情线**：必须穿插主角收服各路美女（女主）的过程，或者英雄救美、暧昧互动的情节。
4. **目标明确**：从微末崛起，最终登顶世界巅峰，拥有无尽财富和权力。
```

### 3.3 章节细纲生成
**作用**：将大纲拆解为具体到每一章的剧情动作。
```text
请根据主线大纲，规划前 {{count}} 章的详细细纲。

【主线大纲】
{{plotContext}}

【人物】
{{charContext}}

要求：
1. **黄金三章法则**：
   - 第一章：必须有强烈的冲突（被退婚、被羞辱、重生节点），并结尾觉醒金手指。
   - 第二章：小试牛刀，利用金手指获得初步优势，震惊路人。
   - 第三章：第一个小高潮，打脸反派（或反派的小弟），收获第一位美女的好感。
2. **爽点密集**：每一章都要有一个明确的"爽点"（Cool Point）或"期待感"（Hook）。
3. **细致**：summary 字段要包含具体发生的事件，不要只写空话。
```

### 3.4 记忆压缩（剧情回顾生成）
**作用**：防止 AI 遗忘前文，提取关键信息作为长期记忆。
```text
你是一位网文编辑。请阅读以下小说前 {{endChapterIndex + 1}} 章的内容，生成一份**详细的剧情回顾/记忆摘要**。
这份摘要将作为 AI 继续创作后续章节的"长期记忆"。

【重点关注人物】
{{protagonistNames}}

【输入内容】
{{contextText}}

【输出要求】
1. **人称**：使用第三人称。
2. **核心剧情**：概括主角的等级变化、获得的关键宝物、击败的主要敌人。
3. **情感进度（至关重要）**：
   - 必须逐一列出主角与每一位已登场女主的互动进展。
   - 她们目前对主角的好感度如何？发生了什么暧昧或冲突事件？
4. **未完伏笔**：列出目前剧情中挖了坑但还没填的伏笔。
5. **格式**：分点叙述，条理清晰，字数控制在 1000 字以内。
```

### 3.5 正文写作（流式生成）
**作用**：最终的文字输出，要求极高的网文语感。
```text
你现在是全网最火的爽文作家。请根据以下大纲撰写第 {{chapterNumber}} 章的正文。

【章节信息】
标题：{{title}}
梗概：{{summary}}
爽点：{{coolPoint}}
核心规则：{{rules}}

【前情提要 / 长期记忆】
{{storySummary}}

【上下文】
上一章内容：{{prevContext || "这是第一章，直接开始"}}

【世界与人物】
世界风格：{{worldTheme}}
人物设定：
{{charContext}}

【写作要求】
1. **文风**：小白文风格，短句为主，节奏极快。不要文青，不要谜语人。
2. **感官描写**：
   - 对女性角色：重点描写身材（长腿、细腰、曲线）、五官、声音（娇喘、冷哼）和气味（体香）。
   - 对打脸情节：描写反派的嚣张嘴脸（前）和震惊、恐惧的表情（后），形成强烈对比。
   - 对路人反应：要有大量的"倒吸一口凉气"、"目瞪口呆"等侧面描写来衬托主角的牛逼。
3. **对话**：要口语化，装逼台词要够硬，撩妹台词要够骚。
4. **字数**：2000字左右。
5. **排版**：段落之间空一行，方便手机阅读。

【特别修订指令】
{{additionalInstructions}}
```

## 4. 详细调试与开发指南 (Debugging Guide)

对于想要修改或调试代码的小白开发者，请参考以下指南：

### 4.1 如何查看 AI 的真实返回？
在开发过程中，如果 AI 生成失败，按 `F12` 打开浏览器的**开发者工具 (DevTools)**，切换到 **Console (控制台)** 标签页。
代码中已经埋入了详细的日志：
```typescript
// 示例：在 services/geminiService.ts 中
console.log(`正在请求 ${provider} 生成后宫爽文人设和世界观...`);
// 如果报错，控制台会打印：
console.error("生成角色失败:", error);
```

### 4.2 API 请求受限 (429 Too Many Requests) 怎么办？
大模型 API 通常有频率限制（比如一分钟只能请求 15 次）。
**解决方案**：我们在 `services/geminiService.ts` 中编写了 `retryOperation` 函数。
```typescript
// 带有固定时间间隔的重试包装函数
// 策略：失败后等待 2秒 -> 10秒 -> 30秒 -> 60秒 再重试
async function retryOperation<T>(operation: () => Promise<T>): Promise<T> {
    // ... 捕获 429 错误并自动 sleep 等待 ...
}
```
如果你发现界面一直处于 Loading 状态，请查看 Console，可能会看到 `[AI Service] API 请求受限或繁忙，将在 10秒后重试...` 的提示。请耐心等待即可。

### 4.3 如何彻底清除本地缓存（重置项目）？
如果你发现页面卡死，或者想彻底清空旧的小说数据：
1. 按 `F12` 打开开发者工具。
2. 切换到 **Application (应用)** 标签页。
3. 在左侧找到 **Local Storage (本地存储)** -> 点击你的网站域名。
4. 找到 Key 为 `soulwriter_autosave_v1` 的数据，右键点击 **Delete (删除)**。
5. 刷新页面，项目将恢复到初始状态。

### 4.4 如何修改 AI 的输出格式？
如果你想让 AI 多生成一个字段（例如给角色增加一个“隐藏身份”字段）：
1. 打开 `types.ts`，在 `Character` 接口中添加 `hiddenIdentity: string;`。
2. 打开 `services/geminiService.ts`，找到 `generateCharactersAndWorld` 函数。
3. 在 `googleSchema` 和 `responseSchemaObj` 中，添加对应的属性定义：
   ```typescript
   // 增加 JSON Schema 定义，强制 AI 输出该字段
   hiddenIdentity: { type: Type.STRING, description: "角色的隐藏身份" }
   ```
4. 在 Prompt 中明确告诉 AI 需要生成这个字段。
